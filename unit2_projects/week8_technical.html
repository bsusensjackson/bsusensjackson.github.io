Intro to Recursion --

I find recursion to be one of the most interesting concepts in programming, and yet it is also one of the hardest to know when to implement. In this blog I'm going to try and break down how recursion works, using Ruby to demonstrate. I must give credit David Black for the first example, if you want to learn Ruby buy his book The Well Grounded Rubyist.

Recursion is the process of a method calling on itself. What is interesting about recursion is that it shows the differences between self and local scope: with recursion there is no change to self, but the variables that you are working with are reset. Let's take a look at the following code:

>class C
>def x(value_for_a, recurse=false)
> a = value_for_a
> print "Vale of 'a' for 'self' :"
> p self
> puts "And here's a:"
> puts a
> if recurse == true
> puts "Calling myself..."
> x("Second value for a")
> puts "Back after recursion: here's a:"
> puts a
> end
>end
end
c = C.new
c.x("First value for a", true)

This will output:
Here's the value for 'self' : #
And here's a:
First value for a
Calling myself..."
Here's the value for 'self' : #
And here's a:
Second value for a
Back after recursion: here's a:
First value for a

What you'll notice from this is that the instance method C#x takes two arguments, the value for a and a boolean flag (which prevents this from becoming an infinite recursion loop.) As we make our way through the lines of the method, we get to the if recurse, which will evaluate to true the first time around (since we passed the method the value true the first time around. (You'll notice that the second method call for x has no value for the second argument, thus defaulting to false.) As we make it through the second call of the method, the values of a changes, but the self doesn't change. Pretty cool stuff right?
